```
cd packages/db && npx tsx migrate-to-ulid.ts
```

The script migrates all primary keys and foreign keys to ULID format:

1. Reads all IDs and `created_at` timestamps upfront (parallel fetches)
2. Generates new ULIDs in TypeScript using `ulid(createdAt.getTime())` so the time component reflects the original creation time
3. Inside a single transaction:
   - Drops all FK constraints and the `user_group` composite PK
   - Bulk-rewrites every FK reference column (one VALUES-based UPDATE per column)
   - Bulk-rewrites all PK `id` columns
   - Recreates all constraints

**What is not migrated:**

- `push_subscription.endpoint` — already a meaningful text PK (push endpoint URL)
- `user_group` — has no own id column; only its `user_id` / `group_id` FK columns are rewritten

**Notes:**

- The entire migration runs in a single transaction — it either fully succeeds or fully rolls back
- After running, `pnpm db push` should report no changes
- New rows created after this migration will use ULIDs generated by the `ulid` npm package in application code
